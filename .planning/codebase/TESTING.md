### 1.2 Fixture Patterns

**Module-scoped fixtures** set up the application for an entire test file:

```python
@pytest.fixture(scope="module")
def app(csv_path):
    os.environ["ABSA_DATA_PATH"] = csv_path
    import main
    main.DATA_FILE_PATH = csv_path
    main.CONFIG_DATA = {"sentiment_elements": [...], ...}
    yield TestClient(main.app)
```

**Function-scoped autouse fixtures** reset global state between tests:

```python
@pytest.fixture(autouse=True)
def reset_config(app):
    import main
    main.CONFIG_DATA["model_a_provider"] = None
    yield
```

Key fixtures:
- `csv_path` — Creates a temporary CSV with test data
- `app` — Returns `TestClient(main.app)` after setting env vars
- `reset_config` — Clears per-test config mutations

### 1.3 Testing Patterns

**Env var chicken-and-egg:** `main.py` reads `ABSA_DATA_PATH` at import time. Module-scoped fixtures set the env var **before** importing main.

**Mock at the highest level:** Endpoint tests mock `services.llm_providers.get_provider` rather than individual provider HTTP calls. This single mock covers the entire dispatch chain.

```python
with patch('services.llm_providers.get_provider') as mock_get_provider:
    mock_provider = MagicMock()
    mock_provider.predict.return_value = ({"labels": [...]}, [])
    mock_get_provider.return_value = mock_provider
    response = app.get(f"/live_prediction/0?role=model_a")
```

**Config mutation:** Tests mutate `main.CONFIG_DATA` directly (not `load_config()`), matching what the Settings panel does.

### 2.2 Test Infrastructure

**React 19 + jsdom workaround:** `@testing-library/react` v16.3.2 crashes on React 19.2.7 CJS bundle (`React.act` is undefined). All component tests use a custom `render()` helper:

```typescript
import ReactDOMClient from 'react-dom/client';
import ReactDOM from 'react-dom';

function render(el: React.ReactElement): { container: HTMLElement; root: ReactDOMClient.Root } {
  const container = document.createElement('div');
  document.body.appendChild(container);
  const root = ReactDOMClient.createRoot(container);
  ReactDOM.flushSync(() => { root.render(el); });
  return { container, root };
}
```

**DOM traversal helper (replaces `screen.getByText`):**

```typescript
function findByText(container: HTMLElement, pattern: RegExp | string): Element | null {
  const walker = document.createTreeWalker(container, NodeFilter.SHOW_TEXT, null);
  let node: Text | null;
  while ((node = walker.nextNode() as Text | null)) {
    if (typeof pattern === 'string'
      ? node.textContent?.includes(pattern)
      : pattern.test(node.textContent || '')) {
      return node.parentElement;
    }
  }
  return null;
}
```

**Native event dispatch (replaces `fireEvent`):**

```typescript
function click(target: Element | null) {
  if (!target) throw new Error('Cannot click null element');
  ReactDOM.flushSync(() => {
    target.dispatchEvent(new MouseEvent('click', { bubbles: true, cancelable: true }));
  });
}
```

**Polyfills needed:**

```typescript
// jsdom doesn't implement scrollIntoView
Element.prototype.scrollIntoView = vi.fn();
```
