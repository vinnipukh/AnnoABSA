## Component Analysis

### ModelTripletColumn (`frontend/src/components/ModelTripletColumn.tsx`)

**Props interface:**

```typescript
interface ModelTripletColumnProps {
  title: string;
  subtitle?: string;
  badgeText?: string;
  badgeColor?: string;           // default: "bg-primary/20 text-primary border-primary/30"
  triplets: TripletItem[];
  selectedIds: Set<string>;
  onToggleSelect: (id: string) => void;
  onSelectAll: () => void;
  onClearAll: () => void;
  // Phase 4: Live Compare Mode
  onRunPrediction?: () => void;  // optional — only for Live mode
  isPredicting?: boolean;        // default: false
}
```

**Rendering pattern:**
- Outer card: `flex flex-col h-full bg-base-200/80 border border-base-300 rounded-2xl p-4 shadow-xl backdrop-blur-sm overflow-hidden`
- Header row with title, optional badge, and select-all/clear-all button
- Scrollable list area with `flex-1 overflow-y-auto space-y-2.5 pr-1 custom-scrollbar`
- Empty state: two variants — (1) with `onRunPrediction` shows a "Run Prediction" button, (2) without shows "No output" with folder SVG icon
- Each triplet: clickable card `div` with `onClick={() => onToggleSelect(t.id)}`
- Card selected state uses sentiment-colored borders/shadow (`cardSelected` / `cardHover` from `getSentimentStyle`)
- Checkbox: custom diamond (◆) inside a colored square, not CustomCheckbox component
- Triplet display: aspect_term in quotes, sentiment badge (colored via `getSentimentStyle`), aspect_category in monospace label
- Selected indicator: vertical accent bar (`absolute left-0 top-3 bottom-3 w-1 bg-primary rounded-r-full`)

**Key observations for 4-way grid adaptation:**
- The component is tightly coupled to a 2-column layout pattern (Model A / Model B)
- Each triplet renders as a full card (~80-100px height) — too tall for 4 columns
- Selection uses `Set<string>` for `selectedIds` — same pattern works for 4 columns
- Sentiment color coding (`getSentimentStyle`) returns text/badge/card/cardHover/icon — can reuse for compact chips
- The select-all/clear-all pattern is per-column, which maps well to 4 columns
- `badgeColor` prop is used to differentiate Model A (secondary colors) vs Model B (accent colors)
- The inline `getSentimentStyle()` function is local to this component and must be re-extracted if needed elsewhere

**Sentiment color scheme used:**
| Polarity | text | badge | cardSelected | icon |
|----------|------|-------|-------------|------|
| positive/pos/olumlu | `text-success font-medium` | `bg-success/10 text-success border-success/30` | `bg-success/5 border-success/60 shadow-success/10` | ◆ |
| negative/neg/olumsuz | `text-error font-medium` | `bg-error/10 text-error border-error/30` | `bg-error/5 border-error/60 shadow-error/10` | ◆ |
| neutral/others | `text-warning font-medium` | `bg-warning/10 text-warning border-warning/30` | `bg-warning/5 border-warning/60 shadow-warning/10` | ◆ |

- Reusable for Ground Truth display in review header (annotation highlighting)

## Data Flow

### CSV → Backend → API → Frontend

```
CSV file (annotations.csv or semeval_tr_llm_annotated.csv)
  → app.data.load_data() (reads CSV via pandas or JSON)
  → app.routes.reviews:get_data(idx) (routes/reviews.py:13-94)
    → builds response dict with:
      - id, text, review_text, label, translation, aspect_category_list
      - model_a_triplets (from "aspect_triplets" column or compare_model_a_csv)
      - model_b_triplets (from "new_triplets" column or compare_model_b_csv)
      - model_a_name, model_b_name (from CONFIG_DATA)
      - agent_initial_reasoning (from data or generated mock)
  → Frontend fetch(`${backendUrl}/data/${index}`)
    → App.tsx:loadReviewRow() (line 166-193)
      → Sets currentData, selectedIds, manualTriplets
      → Falls back to FALLBACK_DATA if API offline
  → React state flows to ModelTripletColumn and ManualInputForm via props
```

### Comparison CSV loading

- `_load_comparison_csv(csv_path, data_idx, review_text, prefix)` in `app/data.py:85-123`
- Two auto-detected formats:
  1. **STD format** (columns: `review`, `triplet`): matched by review text, uses `parse_triplet_column()`
  2. **Per-row format** (columns: `review_id`, `aspect_term`, `aspect_category`, `sentiment_polarity`): matched by numeric index
- Called from `get_data()` when `CONFIG_DATA` has `compare_model_a_csv` or `compare_model_b_csv` set

### parse_triplet_column (`app/data.py:32-82`)

- Accepts: `None`, `"nan"`, ``"[]"``, Python list literal strings, or actual lists
- Three sub-formats:
  1. STD tuples: `[('term', 'CATEGORY', 'polarity'), ...]`
  2. STD lists: `[['term', 'CATEGORY', 'polarity'], ...]`
  3. Dict format: `[{'aspect_term': ..., 'aspect_category': ..., 'sentiment_polarity': ...}, ...]`
- Auto-generates IDs: `{prefix}_{i}` (e.g., `ma_0`, `mb_1`)

### Extension points for NEWUI CSV:

The `semeval_tr_llm_annotated.csv` has columns:
- `text` — review text
- `original_label` — Ground Truth triplets
- `gemma4_31b_label` — Gemma 31B triplets
- `qwen3.6_35b_label` — Qwen 35B triplets
- `gpt_oss_120b_label` — GPT-OSS 120B triplets
- `majority_vote` — integer (3, 2, or 1)
- `majority_label` — majority vote triplets
- `consensus_intersection` — intersection of all 4 model outputs
- `original_llm_diff` — diff between original and majority

The existing `get_data()` endpoint needs to detect NEWUI columns and add `gt_triplets`, `gemma_triplets`, `qwen_triplets`, `gpt_triplets`, `majority_vote`, `majority_label`, `consensus_intersection`, `original_llm_diff` to the response.

## Reusable Patterns

### Directly reusable:
1. **Sentiment color palette**: `getSentimentStyle()` pattern in ModelTripletColumn — text-success/error/warning, badge bg colors, border colors
2. **Selection state pattern**: `Set<string>` for `selectedIds` — simple and efficient, works for 4 columns
3. **Mode switching pattern**: `settings.compare_mode` with string values — adding `'4way'` follows existing precedent
4. **Data fetching pattern**: `loadReviewRow()` async fetch → set state → error fallback
5. **Review text display**: `ManualInputForm`'s review text block is reusable as a standalone review header component
6. **Color engine**: `phraseColoring.tsx` `getColorByIndex()`, `createTextHighlights()`, `renderHighlightedText()` — for Ground Truth annotations
7. **Save-next pattern**: `handleNextReview()` — collects approved triplets, POSTs to backend, advances

### Needs adaptation:
1. **ModelTripletColumn** → Needs compact chip rendering (single-line per triplet, not full card). Current cards are ~80-100px tall; chips should be ~28-36px
2. **ManualInputForm** → Needs to be split: review text section becomes standalone review header, form section moves to resolution panel
3. **Live mode state** → `liveModelATriplets`/`liveModelBTriplets` parallel state — for 4-way, need `liveGemmaTriplets`, `liveQwenTriplets`, `liveGptTriplets` (or a dictionary)
4. **Select-all/clear-all** → Currently two pairs (Model A / Model B). Need 4 pairs or a generic `toggleColumn(column, ...)` pattern
5. **AppActions** → `selectTriplet(role, id)` uses `'model_a' | 'model_b'` — needs extension for `'gt' | 'gemma' | 'qwen' | 'gpt'`
6. **Save endpoint** → `POST /review/{idx}/save` sends `triplets: [...]` — NEWUI needs to save tier metadata (which column was selected, majority vote, etc.) or just the approved triplets as before

## Risk Areas

### 1. Tight coupling in App.tsx (HIGH)
- `toggleModelA`/`toggleModelB` are hardcoded for exactly 2 models
- `selectedModelAIds`/`selectedModelBIds` are separate useState calls — will need 4 for 4-way or a refactored dictionary pattern
- Live mode state (`liveModelATriplets`, `liveModelBTriplets`) similarly hardcoded for 2 models
- **Mitigation**: Use a dictionary keyed by column ID, e.g. `selectedIds: Record<string, Set<string>>` or a flat generic `toggleColumn(column, id)` pattern

### 2. Mode complexity (MEDIUM)
- Currently 2 modes (compare / manual) × 2 sub-modes (csv / live) = 4 states
- Adding `'4way'` makes it 2 × 3 = 6 states
- The render block (lines 631-692) uses a ternary `mode === 'compare'` — adding 4-way as a sub-check inside compare increases nesting
- **Mitigation**: Extract compare mode rendering into a sub-component that handles csv/live/4way internally

### 3. Backward compatibility (MEDIUM)
- Existing CSV files without NEWUI columns must continue to work in "Standard" mode
- The `compare_mode` setting currently defaults to `'csv'` — this must remain for existing users
- The backend must auto-detect NEWUI columns and switch to 4-way mode, or respect the user's sub-mode choice

### 4. Resolution panel placement (MEDIUM)
- Current compare layout uses `grid-cols-3` for [Model A | ManualForm | Model B]
- New layout needs left ~70% for 2x2 grid, right ~30% for resolution panel
- The footer (save & next) and floating chat must not overlap with the panel
- NLP Toolbar positioning must be checked (requirement NEWUI-10)

### 5. Test patterns (LOW)
- Tests use vitest + react-dom without testing-library (React 19.2.7 known CJS bug)
- New components must follow the same pattern: `render()` helper, `findByText()`, `click()`

### 6. Data model extension (LOW-MEDIUM)
- `ReviewComparisonData` currently has exactly `model_a_triplets` and `model_b_triplets`
- Adding `gt_triplets`, `gemma_triplets`, `qwen_triplets`, `gpt_triplets` makes it complex
- If the field is optional (`undefined` when not in NEWUI mode), existing code referencing `.model_a_triplets` still works via fallback
