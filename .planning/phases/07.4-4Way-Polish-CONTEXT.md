<decisions>
## Implementation Decisions

### D-01 (NEWUI-11 ‚Äî Column header names)
Show the CSV column name (e.g. `gemma4_31b_label`) on each 2x2 grid card header alongside the short display name. The column name is available from the CSV data through the existing `NEWUI_COLUMNS` mapping. Add a small monospace/subtle label below or beside the model name.

### D-02 (NEWUI-12 ‚Äî Demo mode)
**Decision:** Frontend `FALLBACK_DATA` array (same pattern as `useReviewNavigation.ts`).
- Create a demo data set covering all 3 tiers.
- A 4th toggle button in the mode selector (Compare / Manual / 4-Way / Demo).
- Demo mode loads reviews from the frontend's fallback data instead of the backend.
- The toggle should be labeled "Demo" and visually distinct.
- When activated, it loads demo reviews into the 4-way grid.
- Condition: only available when no backend data is loaded, or explicitly toggled.

### D-03 (NEWUI-13 ‚Äî Tier filter dropdown)
**Decision:** Filter queue ‚Äî only navigate through reviews matching the selected tier.
- Dropdown options: "All" (default), "Tier 1", "Tier 2", "Tier 3".
- When a specific tier is selected, `handleNextReview`/`handlePrevReview` skip reviews that don't match.
- The current index jumps to the next matching review.
- Visual indicator shows the current filter (e.g. "Tier 2 filtreleniyor").

### D-04 (NEWUI-14 ‚Äî Auto-save on navigation)
**Decision:** Save immediately on navigation + toast confirmation.
- `handleNextReview` and `handlePrevReview` auto-save selected + manual triplets before navigating.
- Toast: "ƒ∞nceleme #N kaydedildi (M etiket)" ‚Äî already partially implemented in Phase 7.3.
- The existing flow already saves on `handleNextReview` ‚Äî extend `handlePrevReview` to do the same.

### D-05 (NEWUI-15 ‚Äî Save button in resolution panel)
**Decision:** Both ‚Äî save button next to tier badge in resolution panel header + auto-save on navigation.
- Add a "üíæ Kaydet" button (using Heroicons SVG save icon, NOT emoji) visible in the resolution panel header.
- The button triggers `saveAndNext()` or just `saveReview()` without advancing.
- Auto-save on navigation handles the advance case.
- The save button is for users who want to save mid-review without navigating away.

### D-06 (NEWUI-16 ‚Äî CSV export)
**Decision:** All columns ‚Äî original CSV columns + annotation results + selected triplets.
- New backend endpoint: `GET /data/export-4way`.
- Returns the original data with appended columns:
  - `selected_triplets` ‚Äî JSON array of selected triplet IDs per review
  - `resolution_tier` ‚Äî the assigned tier (1/2/3)
  - `annotator_notes` ‚Äî empty column for researcher notes (or prepopulated with resolution outcome)
- Frontend download button in the resolution panel or toolbar area.
- Format: CSV download with proper headers.

</decisions>

<specifics>
## Specific Ideas

- **Tier filter**: Add a `tierFilter` state in App.tsx ("all" | 1 | 2 | 3). The navigation functions check `currentData.majority_vote` against the filter and skip non-matches. Use a loop with max retries (e.g. 50) to prevent infinite loops on empty filters.
- **Demo toggle**: A 4th button next to Compare/Manual/4-Way. When active, `currentData` is loaded from the demo array instead of the backend. The demo array should have ~6 reviews covering all tier scenarios.
- **Save button**: A DaisyUI `btn btn-sm` with a Heroicons save SVG in the resolution panel header. No emoji.
- **Export endpoint**: Reads the current data file, applies any in-memory annotations (selected triplets), and returns as CSV download. Use `StreamingResponse` with `media_type="text/csv"`.
</specifics>
