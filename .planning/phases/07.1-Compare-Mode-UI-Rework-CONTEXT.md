# Phase 7.1: Compare Mode UI Rework - Context

**Gathered:** 2026-07-13
**Status:** Ready for planning

<domain>
## Phase Boundary

Redesign the Compare mode from a 2-column (Model A / Model B) layout to a 4-way 2x2 grid with consensus resolution panel and 3-tier curation workflow. Ingests `semeval_tr_llm_annotated.csv` with 4 LLM columns + Ground Truth + consensus metadata. NEWUI mode is a sub-toggle within Compare mode, alongside Standard (old CSV) and Live modes.

</domain>

<decisions>
## Implementation Decisions

### Layout & Grid
- **D-01 (Layout approach):** Narrow/compact triplet chips to fit all 4 columns at once (1280px+ target). Each column renders triplets as single-line chips rather than full card blocks.
- **D-02 (Resolution panel):** Fixed right-side panel as specified in NEWUI-01. Grid takes left ~70%, resolution panel takes right ~30%.
- **D-03 (Collapsible cards):** Rejected — no collapsible/minimizable cards. All 4 columns are always visible. Compact rendering makes this feasible.
- **D-04 (Review header):** Remains at the top of the page, unchanged from current layout.

### Backend Data Model
- **D-05 (Endpoint strategy):** Extend existing `GET /data/{idx}` endpoint rather than creating a new one. When the loaded CSV is `semeval_tr_llm_annotated.csv` with NEWUI columns, the response includes additional fields for the 4 model columns and consensus metadata.
- **D-06 (Response fields):** Add `gt_triplets`, `gemma_triplets`, `qwen_triplets`, `gpt_triplets`, `majority_vote`, `majority_label`, `consensus_intersection`, `original_llm_diff` to the response. Existing `model_a_triplets`/`model_b_triplets` and `label` remain for backward compatibility.

### Mode Integration
- **D-07 (Mode hierarchy):** NEWUI is a sub-toggle within Compare mode. The toggle becomes: **Standard** (old CSV) / **4-Way** (NEWUI) / **Live**. No new top-level mode. Backend config key `compare_mode` gets `'4way'` as a new value option.
- **D-08 (Old CSV compare preserved):** The Standard option retains the existing 2-column CSV compare mode exactly as-is. Users can switch back.

### Curation Tiers
- **D-09 (Tier 1 — Auto-Accept, green):** Pre-selects Ground Truth (`original_label`) triplets automatically since they match the majority. User can simply save-and-next without interaction. Badge shows green with checkmark.
- **D-10 (Tier 2 — Quick Diff, yellow):** Diff Tracker shows a side-by-side comparison of original vs. majority labels. User manually picks which triplets to accept from either side. Badge shows yellow with warning icon. Diff Tracker box is highlighted with a subtle glow/border.
- **D-11 (Tier 3 — High-Confusion, red):** Resolution panel shows all 4 models' outputs for side-by-side comparison. `majority_label` is hidden. User performs manual review across the full 2x2 grid. Badge shows red with exclamation. User creates triplets manually via existing ManualInputForm or inline controls.
- **D-12 (Consensus badge):** Green circle at center of 2x2 grid showing `majority_vote` integer. Color-coded: Green=3, Yellow=2, Red=1. Matching the tier colors.

### Claude's Discretion
- Resolution panel action button design (accept/edit/manual) — implement standard save/discard/edit patterns
- Diff Tracker rendering style — use monospaced diff-style as specified in NEWUI-05, exact visual treatment open
- Responsive breakpoint below 1280px — handle gracefully with overflow
- Keyboard shortcuts for tier actions
- Transition/animation between review items

</decisions>

<code_context>
## Existing Code Insights

### Reusable Assets
- **ModelTripletColumn.tsx** — Existing column component with title/badge/subtitle, select-all/clear-all, per-triplet checkbox, sentiment color coding. Can be adapted for compact chip rendering.
- **CustomCheckbox.tsx** — Reusable styled checkbox component used by ModelTripletColumn.
- **phraseColoring.tsx** — Color engine with 25 Tailwind color classes, character-level highlight rendering. Ground Truth display can reuse this.
- **useTextSelection.ts** — DOM Range-based text selection hook, should remain intact for review header annotation.

### Established Patterns
- **Two-process architecture:** Backend FastAPI + frontend React/Vite. REST-only communication.
- **Global state in App.tsx:** All UI state managed via useState, no Redux/Context. NEWUI state should follow same pattern.
- **Mode switching:** Current `compare_mode` key in settings controls CSV/Live sub-mode. Adding `'4way'` follows existing pattern.
- **CSV comparison loading:** `_load_comparison_csv()` in data.py loads per-row CSV comparison data. NEWUI data can use a similar loader.

### Integration Points
- Backend: `app/routes/reviews.py:get_data()` — extend to detect NEWUI CSV columns and return additional fields
- Backend: `app/data.py:_load_comparison_csv()` — model NEWUI loader after this existing function
- Backend: `app/config.py:CONFIG_DATA` — add `compare_mode: '4way'` support
- Frontend: `App.tsx` mode/toggle logic — extend compare sub-mode to include 4-Way
- Frontend: `types.ts:ReviewComparisonData` — add NEWUI fields
- Frontend: `types.ts:Settings` — add `compare_mode: 'csv' | 'live' | '4way'`

</code_context>

<specifics>
## Specific Ideas

- Compact triplet rendering: each triplet shown as a single horizontal chip with aspect_term (truncated), sentiment badge (colored dot or short label), and category tag. No full card layout.
- Resolution panel: Primary Suggestion Box shows `majority_label` (for Tier 1/2) or "No consensus — compare models below" (for Tier 3). Diff Tracker Box shows `original_llm_diff` with monospaced diff-style rendering.
- Consensus badge positioned at the center of the 2x2 grid, overlapping the four card corners with a circular badge.
