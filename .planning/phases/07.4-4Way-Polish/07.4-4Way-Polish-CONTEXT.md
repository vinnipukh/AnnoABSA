# Phase 7.4: 4-Way Polish — Context

**Gathered:** 2026-07-13
**Status:** Ready for planning

<domain>
## Phase Boundary

Polish the 4-Way Compare mode with: visible CSV column names on grid cards, a Demo mode with pre-built sample cases for all 3 tier scenarios, a tier filter dropdown for review navigation, auto-save on review transition, a visible save button in the resolution panel, and a CSV export for annotated 4-way outputs.
</domain>

<decisions>
## Implementation Decisions

### Column Header Names
- **D-01 (Column display):** Each 2x2 grid card header shows the model short name (GT, Gemma, Qwen, GPT) AND the actual CSV column name as subtitle (e.g. `original_label`, `gemma4_31b_label`, `qwen3.6_35b_label`, `gpt_oss_120b_label`).
- **D-02 (Backend field):** Add `model_column_names` field to GET /data/{idx} response for 4-way mode, mapping column keys to display names.

### Demo Mode
- **D-03 (Demo activation):** A 4th toggle button in the sub-mode row: **Standard / 4-Yönlü / Canlı / Demo**. Only visible when in Compare mode.
- **D-04 (Demo data source):** When Demo is clicked, injects a pre-built sample dataset from the frontend (no backend CSV needed) containing rows that cover:
  - Tier 1 (majority_vote=3, majority matches GT)
  - Tier 2 (majority_vote=2, majority differs from GT)
  - Tier 3 (majority_vote=1)
  - Edge case: Empty/missing triplets in some columns
- **D-05 (Demo behavior):** The 4-Way grid + resolution panel work identically on demo data. User can explore tiers freely. No save needed in Demo mode.

### Tier Filter
- **D-06 (Filter type):** Dropdown above the 2x2 grid: **All Reviews** / **Tier 1 (Auto-Accept)** / **Tier 2 (Quick Diff)** / **Tier 3 (Manual)**.
- **D-07 (Filter behavior):** Filters the review queue to only show reviews matching the selected tier. Next/prev navigation jumps within filtered set. Current index resets to first match.
- **D-08 (Filter source):** Frontend computes tier from `majority_vote` and `gt_triplets` vs `majority_label` comparison (same logic as ResolutionPanel's tier detection).

### Auto-Save
- **D-09 (Auto-save trigger):** Auto-saves when navigating to next/previous review (save & advance becomes automatic). Does NOT save on every single triplet selection change.
- **D-10 (Save confirmation):** Show a brief toast notification on auto-save: "✅ Kaydedildi" (Saved). Toast auto-dismisses in 2 seconds.
- **D-11 (Manual save):** Keep the existing save & next button in footer. Also add a visible **Save** button inside the resolution panel.

### Save Button in Resolution Panel
- **D-12 (Save button placement):** Below the action buttons (Accept/Edit/Manual), always visible. Shows "💾 Kaydet ve İlerle" (Save & Next).
- **D-13 (Save behavior):** Triggers the same save endpoint as the footer button — collects selected triplets from all columns, saves via POST /review/{idx}/save, advances to next review.

### CSV Export
- **D-14 (Export endpoint):** New backend endpoint `GET /data/export-4way` that writes a clean CSV file with columns: `review_text`, `gt_triplets`, `selected_triplets`, `majority_vote`, `tier_label`, `save_timestamp`.
- **D-15 (Export button):** A button in the header/4-way view labeled "📥 Çıktıyı İndir" that triggers the export endpoint and downloads the CSV.
- **D-16 (Export scope):** Exports only reviews that have been saved (non-empty label). Includes the `majority_vote` and computed tier for traceability.

### Claude's Discretion
- Toast styling and positioning for auto-save feedback
- Demo dataset content (which reviews, which terms to use)
- Export file naming convention
- Filter dropdown styling

</decisions>

<specifics>
## Specific Ideas

- Demo data: Create a static array of 4-5 sample reviews in the frontend with all tier scenarios hardcoded. No backend call needed.
- Filter: Use a simple `useMemo` to filter the review indices based on tier, then pass filtered indices to the navigation.
- Auto-save: Wrap the existing `handleNextReview()` / `handlePrevReview()` to call save first, then navigate.
- Export: Backend endpoint reads the current data file, checks which rows have non-empty `label`, builds a clean CSV, returns it as a download.

</specifics>
