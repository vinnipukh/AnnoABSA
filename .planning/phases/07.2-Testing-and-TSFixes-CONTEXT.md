<decisions>
## Implementation Decisions

### Hardening Scope
- **D-01 (tsconfig target):** Change `target` from `"es5"` to `"es2016"` in `tsconfig.json`. This alone fixes the `includes` error (TSFIX-03). Also update `lib` to `["dom", "dom.iterable", "es2016"]`.
- **D-02 (Vite upgrade):** Upgrade Vite from `^4.5.14` to latest v5.x to eliminate the `esbuild`/`oxc` deprecation warnings. Update `vite.config.js` to remove deprecated `esbuild` and `optimizeDeps.esbuildOptions` config blocks.
- **D-03 (App.tsx split):** Extract state + handler logic into custom hooks. The split follows a clean separation:

  | Hook | State extracted | API calls |
  |------|----------------|-----------|
  | `useReviewNavigation` | `currentIndex`, `currentData`, `totalCount`, `isLoading` | `fetchData()`, `saveReview()` |
  | `useAnnotationState` | `manualTriplets`, `selectedModelAIds`, `selectedModelBIds` | `saveReview()` |
  | `useAIPrediction` | `aiSuggestions`, `liveModelATriplets`, `liveModelBTriplets`, `isAIPredicting` | `fetchAIPrediction()`, `fetchLivePrediction()` |
  | `useSettings` | `settings`, `saveToast` | `fetchSettings()`, `PATCH /settings` |
  | `useCompareMode` | `mode`, `compareMode` | — (UI state only) |

  Each hook goes in its own file under `frontend/src/hooks/`. `App.tsx` then composes them.

- **D-04 (Hook tests):** Each extracted custom hook gets a corresponding vitest test file. At minimum: `useReviewNavigation`, `useAnnotationState`, `useAIPrediction`, `useSettings` get tests. `useCompareMode` is trivial enough to skip.

### Test Approach
- **D-05 (Fixture strategy):** Use `conftest.py` fixtures for all new backend tests. Each test domain gets its own conftest or shared fixtures for common setup (temp CSV file, temp config file, TestClient).
- **D-06 (Backend test files):**

  | Test file | Target | Approach |
  |-----------|--------|----------|
  | `tests/test_active_learning.py` | `services/active_learning.py` | Pure unit tests for `train_labeled_data()`, `get_uncertainty_scores()`, `labeled_texts_from_data()`. Parametrize edge cases (empty data, single label, mixed polarity). No mocking needed for sklearn pipeline. |
  | `tests/test_learning_routes.py` | `app/routes/learning.py` | FastAPI TestClient with temp CSV fixture. Test `GET /learning/suggestions` and `GET /learning/predict/{idx}` with various data states. |
  | `tests/test_cli.py` | `cli/config.py`, `cli/convert.py`, `cli/runner.py` | Config builder/setter tests. STD format conversion tests. Runner: smoke test port binding (don't start full stack). |
  | `tests/test_routes_misc.py` | `settings.py`, `timing.py`, `upload.py` | TestClient with temp config + data files. Test GET/PATCH settings, timing POST/GET, upload POST. |

- **D-07 (Python test target):** Existing 133 tests + 16+ new tests (4 files × ~4 tests each) + hook tests. Final target ~160+ (below the milestone's 184 but Phase 7.1 and 7.3 new code will push over 184).

### TSFIX
- **D-08 (TSFIX-01 — `rect` error):** Add `rect?: DOMRect` to the `nlpToolbarSelection` state type in `App.tsx`. The state currently only has `{ text, sentence }`, needs `rect` for NLP toolbar positioning.
- **D-09 (TSFIX-02 — `global` error):** Replace `global.fetch = mockFetch` with `vi.stubGlobal('fetch', mockFetch)` in `NlpHelperToolbar.test.tsx:46`.
- **D-10 (TSFIX-03 — `includes` error):** Fixed by D-01 (tsconfig target change). The `includes` method needs `lib: es2016` or later.

<code_context>
## Existing Code Insights

### Reusable Assets
- `tests/test_live_prediction.py` — Has established pattern: `TEST_REVIEWS_CSV` data as module-level constant, `tempfile.NamedTemporaryFile` + `unittest.mock.patch` for config mocking, `TestClient` for endpoint testing. Reuse for TEST-02 and TEST-04.
- `tests/test_prediction.py` — Pure unit test pattern (no mocking needed for algorithmic logic). Reuse for TEST-01.
- `frontend/src/hooks/useTextSelection.ts` — Existing hook pattern (useState + useCallback + useMemo). New hooks should follow same structure.
- `frontend/src/components/HelperAgentChatbox.test.tsx` — Existing component test with `createRoot`+`flushSync` pattern (React 19 workaround). Hook tests won't need this — they test state logic directly.

### Established Patterns
- **Backend tests:** pytest with `import` + `unittest.mock.patch` for external dependencies. `FastAPI TestClient` for endpoint tests.
- **Frontend tests:** vitest with `vi.fn()` mocking. `@testing-library/react` for component rendering. `createRoot`/`flushSync` workaround for React 19 CJS compatibility.
- **Data testing:** Tests use `tempfile.NamedTemporaryFile` with inline CSV/JSON strings to avoid file system dependencies.

### Integration Points
- New `conftest.py` at `tests/` level to provide shared fixtures (temp CSV, config path, TestClient)
- Vite config changes affect `vite.config.js` only — no cascading changes expected
- App.tsx split must preserve the `AppActions` interface that `HelperAgentChatbox` depends on via `useRef`

</code_context>

<specifics>
## Specific Ideas

- tsconfig target change: `"es5"` → `"es2016"`, lib: `["dom", "dom.iterable", "es2016"]`
- Vite upgrade: the `esbuild` and `optimizeDeps.esbuildOptions` blocks in vite.config.js are deprecated and should be removed for v5 compatibility
- App.tsx split: each hook file named after its concern (e.g., `useReviewNavigation.ts`, `useAnnotationState.ts`)
- The `rect` fix in TSFIX-01: the `handleNlpSelectionChange` callback accepts an optional `DOMRect` but the `nlpToolbarSelection` state type only has `{ text: string; sentence: string }`. Adding `rect?: DOMRect` resolves it.

</specifics>

<deferred>
## Deferred Ideas

- **`strict: true` / `noImplicitAny: true`** — Enabling full TypeScript strict mode would surface 30-40 additional type errors. Defer to future hardening phase. The current `strict: false, noImplicitAny: false` is acceptable for a research tool.

</deferred>
