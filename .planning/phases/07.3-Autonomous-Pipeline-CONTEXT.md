<decisions>
## Implementation Decisions

### D-01 (AUTOPILOT-01 — Template actions)
Add the 3 missing actions (`selectTriplet`, `addTriplet`, `annotateAll`) to `DEFAULT_CHAT_TEMPLATE`. No full-template rewrite — the existing 11 actions with Turkish descriptions and `[[action:...]]` examples are correct. Append the new ones.

### D-02 (AUTOPILOT-02 — Keyboard shortcut)
Wire `Ctrl+Shift+L` to toggle the Active Learning suggestions panel. Reuse the existing `keydown` event listener pattern at `App.tsx:247`. The handler checks `e.ctrlKey && e.shiftKey && e.key === 'l'`.

### D-03 (AUTOPILOT-03 — Auto-suggest banner)
**Both** inline banner + toast pattern:
- Inline banner appears above the review header when the current review has high active-learning uncertainty.
- Shows text like "Bu inceleme yüksek belirsizlik taşıyor — etiketlemek ister misiniz?" with an "Etiketle" (Annotate) action button.
- Toast notification dismisses after 4 seconds as a secondary notification.
- Banner has a dismiss (X) button.
- Banner should not appear on reviews that already have labels.

### D-04 (AUTOPILOT-04 — selectTriplet in prompt)
Already exists in `AppActions`. Just add `selectTriplet(role, id)` to the `DEFAULT_CHAT_TEMPLATE` action list. Already wired to `toggleTriplet` in App.tsx:270 — no backend changes needed.

### D-05 (AUTOPILOT-05 — addTriplet signature)
New 3-argument action: `addTriplet(aspect_term, aspect_category, polarity)`. 
- Implemented as a lightweight wrapper in `App.tsx` that constructs a `TripletItem` and calls the existing `addManualTriplet`.
- Registers in `AppActions` and `DEFAULT_CHAT_TEMPLATE`.
- Existing `addManualTriplet(triplet)` remains for programmatic use.

### D-06 (AUTOPILOT-06 — Predictions endpoint)
New dedicated endpoint: `GET /chat/predictions/{data_idx}`.
- Returns chat-formatted text (a string) that the Helper Agent can read, reason about, and act on.
- Not a modification of `GET /learning/predict/{idx}` — keeps concerns separated.
- Response format: `{"text": "...", "predictions": [...]}` — text is a Turkish paragraph summarizing the ML model's predictions.
- The text gets injected into the `initialReasoning` or appended to the chat context.

### D-07 (AUTOPILOT-07 — annotateAll pipeline)
**User confirmation:** TBD (user was undecided — discuss during planning). Options:
- Option A: Execute silently — agent calls `annotateAll()` and it runs without confirmation.
- Option B: Require confirmation via chat — agent says "I'll annotate all" and waits for user approval.

Implementation regardless of confirmation model:
1. `GET /learning/predict/{currentIndex}` to get ML predictions
2. Filter predictions by confidence threshold (default >0.5)
3. For each prediction: call `addTriplet(term, category, polarity)`  
4. Call `saveAndNext()` to persist and advance
5. Repeat from step 1 for the next review
6. Stop after N reviews (configurable, default 5) or when no predictions available

</decisions>

<specifics>
## Specific Ideas

- **Ctrl+Shift+L**: The existing shortcut handler at App.tsx:247 already handles `Ctrl+Shift` combinations. Add a new case for `'l'` that toggles the `showALSuggestions` state.
- **Auto-suggest banner**: A simple `<div>` with DaisyUI `alert alert-info` classes above `<ReviewHeader>`. Conditionally rendered based on `currentData.uncertainty` and `currentData.has_labels`.
- **annotateAll() loop**: Use `async`/`await` with a `for` loop of max N iterations. Each iteration: fetch predictions → add triplets → save → advance. Debounce between iterations (500ms) to let UI update.
- **Chat predictions endpoint**: Wraps the existing ML pipeline from `services/active_learning.py` and formats output as natural Turkish text. No LLM call needed — it's just formatting the sklearn output.
</specifics>
